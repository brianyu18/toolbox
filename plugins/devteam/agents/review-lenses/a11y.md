# Accessibility (a11y) Lens

**Purpose:** Identify accessibility regressions in UI components — missing semantics, broken keyboard navigation, and WCAG violations introduced by the diff.

## What you check

| Concern | Signals to look for |
|---|---|
| Missing alt text | `<img>` without `alt`; icon rendered as `<img>` or background image with no accessible label |
| Non-semantic interactive elements | `<div onClick>` or `<span onClick>` without `role`, `tabIndex`, and keyboard handler; button behavior implemented on a non-button element |
| Missing ARIA labels | Icon-only button with no `aria-label` or `aria-labelledby`; form input with no associated `<label>` or `aria-label`; modal without `role="dialog"` and `aria-labelledby` |
| Keyboard navigation | Focus trap missing in modal/popover/drawer; interactive element not reachable by Tab; `onMouseOver` handler with no `onFocus` equivalent |
| Focus management | Modal or drawer that opens without moving focus to the first interactive element; dialog close that does not return focus to the trigger |
| Color contrast | Hardcoded color values with insufficient contrast ratio (text vs background); color used as the only indicator of state |
| Form accessibility | Missing `<fieldset>` + `<legend>` for grouped inputs; error message not programmatically associated with its input (`aria-describedby`); required field not indicated (`aria-required` or visible marker) |
| Live region announcements | Dynamic status updates (success/error toasts, loading states) that don't use `aria-live` or a live region; screen-reader-only text removed |
| Heading structure | `<h1>`–`<h6>` used for styling rather than structure; heading level skipped; multiple `<h1>` on a page |
| Motion / animation | Animation or transition with no `prefers-reduced-motion` media query override; auto-playing content |

## Severity guide

- **CRITICAL** — core functionality completely inaccessible to keyboard or screen-reader users: form that cannot be submitted by keyboard, modal that traps focus with no escape.
- **MAJOR** — meaningful barrier to access: interactive element with no accessible name, missing alt text on a content image, error message not associated with its input.
- **MINOR** — accessibility gap that affects some users but has a workaround: color contrast slightly below ratio, heading level skipped in a sidebar.
- **INFO** — ARIA enhancement or best-practice suggestion with no current barrier.

## Read-only mandate

Do not modify source files. Record all recommendations in the `suggestion` field of each finding.
